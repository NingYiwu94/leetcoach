# LeetCoach 当前产品说明与下一步路线

本文档用于说明 LeetCoach 当前已经具备的功能、整体产品风格、技术结构和后续推进方向。

LeetCoach 的定位是：

> 一个面向长期刷题的个人算法学习管家，而不是单纯的 AI 题解工具。

它的核心目标不是每天输出很多建议，而是帮用户把“要做的题、已经做过的题、需要复习的题、AI 沉淀下来的题解资产”管理好。

---

## 1. 当前产品形态

当前推荐入口是：

```bash
python coach_app.py
```

`main.py` 仍然保留为备用命令行入口，但主流程已经转向图形界面。

GUI 当前采用左侧导航 + 右侧内容区结构：

- 今天
- 计划
- 题库
- AI 解题
- 同步
- 设置

产品主流程是：

```text
同步力扣记录
→ 生成 / 更新本地学习记录
→ 今日页显示当天任务
→ 用户点击圆圈完成
→ 题库沉淀学习资产
→ 当前阶段完成后生成下一阶段计划草案
→ 用户确认后进入下一阶段
```

---

## 2. 当前核心功能

### 2.1 今天页

今天页是 LeetCoach 的主入口。

当前设计原则：

- 只显示今天要做的任务
- 不显示同步详情
- 不显示本周进度
- 不输出冗长学习建议
- 没任务时显示干净的空状态
- 点击任务前的圆圈即可完成任务

今天页承担的是“任务清单”的职责，而不是分析面板。

它会展示：

- 今日计划题
- 今日需要复习的题
- Day 6 / Day 7 这类阶段任务
- 当前任务完成后的即时状态变化

完成任务后：

- 任务会变成已完成或从今日清单中消失
- GUI 自动刷新
- 必要时后台检查是否需要生成下一阶段计划草案

---

### 2.2 计划页

计划页负责展示当前学习阶段。

当前能力包括：

- 显示当前 Week / Day
- 显示当前专题，例如“链表”“哈希表”
- 显示本阶段任务清单
- 显示每道题完成状态
- 支持 Day 6 复习验收
- 支持 Day 7 阶段总结任务
- 支持 AI 生成下一阶段计划草案
- 用户确认后才会启用新计划

计划推进不强绑定自然周。

如果用户提前完成当前阶段，可以立即确认下一阶段计划，不需要等真实日期进入下一周。

计划生成的硬规则：

- 当前 Week 自动递增
- 标题尽量只显示专题，不写空泛口号
- 已完成题不能作为新题重复安排
- 已完成题只能作为复习或重做安排
- Day 7 点击圆圈后就是完成任务，不再弹出无意义的总结框
- AI 计划必须经过结构校验和质量评估
- AI 失败时使用规则版保守计划兜底

---

### 2.3 题库页

题库页是 LeetCoach 的学习资产中心。

它不是简单题目列表，而是用来沉淀用户做过的题和理解材料。

当前能力包括：

- 按专题展示能力地图
- 每个专题显示完成数 / 力扣题库总量
- 点击专题进入专题详情页
- 专题详情页展示难度完成情况
- 每道题显示完成状态、最近完成时间、AI 题解标识
- 点击题目进入题目详情页

题目详情页当前沉淀：

- 题号和标题
- 完成状态
- 难度
- 最近完成时间
- AI 题解笔记
- 我的笔记
- 提交记录相关信息

AI 题解已经和题库绑定：

- AI 解题生成后会自动保存到对应题目
- 再次进入题目详情页会优先读取已保存题解
- 已生成过的题解不会重复请求模型，除非用户主动重新生成

---

### 2.4 AI 解题

AI 解题当前保留为基础能力，但不是 LeetCoach 的主线。

当前固定输出模板是：

1. 解题思路
2. 完整代码，带详细中文注释
3. 易错点总结
4. 时间复杂度

当前支持：

- 输入题号生成题解
- 选择 Python 或 C++
- 生成结果自动保存到题库题目页
- 已生成题解优先读取缓存
- 失败时不覆盖旧题解
- 生成中、成功、失败状态可见
- 代码区做了更清晰的阅读样式和基础语法高亮

AI 解题的方向是稳定、清楚、可复习，而不是做复杂聊天机器人。

---

### 2.5 同步页

同步页负责把力扣提交记录同步进 LeetCoach。

当前推荐同步方式是：

```text
Chrome 扩展主动读取力扣页面
→ 推送到本机 LeetCoach 同步服务
→ 写入 records.json 和 reviews.json
```

当前同步能力：

- 启动本地同步服务
- Chrome 扩展读取当前登录浏览器中的力扣提交记录
- 本地接收服务接收提交记录
- 自动写入刷题记录
- 自动生成或更新复习记录
- 显示最近同步状态
- 显示最近新增题目
- 显示未映射题目的处理建议

同步安全边界：

- 不保存账号密码
- 不读取或保存 Cookie
- 不做 Cookie 登录
- 不绕过反爬
- 不强依赖公开 GraphQL 接口
- 失败时不影响本地学习数据

同步页当前正在从“调试工具”收敛为“可信状态页”。

目标是让用户一眼知道：

- 本地服务是否运行
- Chrome 扩展最后一次推送是什么时候
- 新增了哪些题
- 哪些题没有映射成功
- 失败时下一步该做什么

---

### 2.6 设置页

设置页用于放低频功能。

当前保留方向：

- 数据校验
- 当前计划信息
- 计划备份
- 计划归档
- LLM 调用日志
- AI 计划评估结果
- RAG 调试信息

设置页不承担日常学习主流程。

后续原则是继续收敛，不把所有历史功能都暴露给普通用户。

## 3. 数据与本地存储

当前继续使用 JSON 文件，不引入数据库。

主要数据文件：

- `data/records.json`：刷题记录
- `data/reviews.json`：复习记录
- `data/problem_bank.json`：题库基础数据
- `data/problem_notes.json`：用户题目笔记和掌握度
- `data/ai_solution_notes.json`：AI 题解笔记
- `data/plan_task_state.json`：计划任务完成状态
- `data/local_push_sync_state.json`：本地同步服务状态
- `data/llm_call_logs.json`：大模型调用日志
- `data/llm_eval_results.json`：AI 计划评估结果
- `data/rag_embeddings.json`：Embedding RAG 缓存
- `data/rag_debug.json`：最近一次 RAG 检索调试
- `data/silent_agent_memory.json`：静默 Agent 后台动作记录

主要配置文件：

- `config/week_plan.json`：当前正式计划
- `config/week_plan_next.json`：待确认计划草案
- `config/topic_catalog.json`：题库专题配置
- `config/leetcode_config.json`：力扣同步配置

---

## 4. AI 与 LLM Layer

LeetCoach 当前已经不再只是“拼 prompt 调 API”。

当前 AI 工程层包括：

- Prompt 模板：`resources/prompts/`
- 输出结构说明：`resources/schemas/`
- LLM 调用日志：`llm_logger.py`
- Prompt 加载：`prompt_loader.py`
- 输出结构校验：`llm_output_validator.py`
- AI 计划质量评估：`ai_plan_evaluator.py`
- 规则版兜底：模型失败时仍能生成保守计划

AI 主要用于：

- 生成题解笔记
- 生成下一阶段计划草案
- 生成周总结或反思材料
- 为计划生成提供可观察、可评估的 PromptOps 实践

当前设计原则：

- AI 不能直接覆盖正式计划
- AI 计划必须等待用户确认
- AI 输出失败不能影响本地学习管理
- AI 解题优先读取缓存，减少等待
- AI 计划必须接受结构校验和质量评分

---

## 5. RAG 当前状态

LeetCoach 已经进入轻量 RAG 阶段。

当前 RAG 不是复杂框架，而是本地学习资产检索：

数据来源包括：

- 题库数据
- 刷题记录
- 复习记录
- 用户笔记
- AI 题解笔记

当前支持：

- 关键词 RAG
- 可选 Embedding RAG
- 本地 JSON 向量缓存
- RAG 调试记录
- AI 解题时参考本地资料
- AI 计划生成时参考本地学习证据

当前 RAG 的作用：

> 让 AI 不只是回答一道孤立的题，而是能参考用户过去做过什么、错过什么、复习过什么、已经保存过什么笔记。

后续 RAG 可以继续升级，但不急于引入大型框架。

---

## 6. 静默 Agent 当前状态

LeetCoach 已经开始进入轻量 Agent 化。

当前 Agent 不是一个会每天长篇输出建议的聊天机器人。

当前实现的是“静默 Agent”：

- 后台检查当前计划是否完成
- 必要时自动生成下一阶段计划草案
- 不自动启用计划
- 不强制跳页面
- 只在真正生成草案时给一句短提示
- 不制造每日噪声

当前记录在：

- `silent_agent.py`
- `data/silent_agent_memory.json`

它的定位是：

> 在后台帮用户减少管理成本，而不是增加新的信息负担。

## 7. 当前整体风格

LeetCoach 当前风格目标是：

```text
Todoist / 滴答清单式任务清单
+ Apple 风格的克制留白
+ ChatGPT / Notion 式的内容阅读卡片
```

视觉原则：

- 简洁
- 少文字
- 少弹窗
- 留白充分
- 分区清楚
- 状态即时可见
- 按钮反馈明确
- 错误提示可理解
- 不堆无用统计

当前视觉语言：

- 左侧浅色导航
- 红色 `L` 作为品牌符号
- 白色 / 米白色背景
- 红色作为主行动色
- 卡片式内容分区
- 轻量边框
- 统一空状态、加载状态和错误状态
- 页面切换时增加轻微过渡，避免右侧内容突兀刷新

当前产品取舍：

- 今天页要极简
- 计划页负责计划
- 题库页负责沉淀
- 同步页负责可信状态
- AI 解题只做标准题解笔记
- 设置页只放低频工具

## 9. 当前已基本完成的目标

目前 LeetCoach 已经实现了最初设想中的主要闭环：

- 每天打开看到今天任务
- 点击圆圈完成任务
- 同步力扣提交记录
- 根据提交生成学习记录
- 根据记录安排复习
- 题库保存做过的题
- AI 解题保存到题库
- 当前阶段完成后生成下一阶段计划草案
- 用户确认后进入下一阶段
- GUI 风格从工具箱逐步收敛为任务清单软件

## 10. 当前仍需打磨的问题

### 10.1 同步稳定性

同步已经可用，但仍是最需要长期打磨的地方。

后续重点：

- 更稳定地感知 Chrome 扩展状态
- 更清楚地显示最后一次推送
- 更清楚地展示新增题目
- 更清楚地提示未映射题目怎么处理
- 减少需要用户理解的技术细节

### 10.2 AI 解题速度与稳定性

当前 AI 解题已经有缓存和质量检查，但远程模型仍可能慢或失败。

后续重点：

- 更清楚的生成中状态
- 更自然的一键重试
- 更快的模型配置
- 更短、更稳定的 Prompt
- 必要时加入流式输出
- 已缓存题解优先展示

### 10.3 AI 计划可信度

AI 计划已经加入硬规则和评估，但仍需要持续校准。

后续重点：

- 明确说明为什么安排这些题
- 显示哪些题因为已完成被排除
- 显示哪些题因为没掌握被安排复习
- 防止退回 Week 1
- 防止标题空泛
- 防止安排过难或无关题

### 10.4 题库资产深度

题库已经从列表变成资产中心，但还可以继续增强。

后续重点：

- 每道题更清楚显示掌握程度
- 我的笔记更好写、更好保存
- AI 题解阅读体验继续优化
- 复习历史更自然地融入题目页
- 已完成题和未完成题视觉区分更清楚

### 10.5 产品交互细节

当前整体界面已经较稳定，但还可以继续做产品级细节。

后续重点：

- 页面切换更顺
- 按钮反馈更细腻
- 卡片层级更统一
- 空状态更克制
- 错误提示更像正式软件
- 减少不必要弹窗

## 11. 下一步建议路线

### 阶段一：继续收敛同步体验

目标：

让同步页只回答三个问题：

1. 服务是否正常
2. 最近一次同步是什么时候
3. 新增了哪些题 / 哪些题失败

不再显示大段调试文本。

### 阶段二：AI 解题体验稳定化

目标：

让 AI 解题像一个可靠的“生成题解笔记”按钮。

重点：

- 已生成直接读缓存
- 生成中状态清楚
- 失败不覆盖旧内容
- 可一键重试
- 生成后自动跳到题库题目页
- 代码区继续增强阅读体验

### 阶段三：计划生成质量闭环

目标：

让用户信任 AI 为什么这么排计划。

重点：

- 计划生成前收集本地题库资产
- 排除已完成新题
- 标记复习题来源
- 显示计划质量检查结果
- 草案必须等待用户确认

### 阶段四：题库成为长期学习资产库

目标：

让用户感觉每道做过的题都没有丢。

重点：

- 题目详情页继续分层
- AI 题解、我的笔记、掌握程度更自然
- 专题页更像能力地图
- 复习记录更简洁地呈现

### 阶段五：谨慎 Agent 化

目标：

Agent 不输出噪声，只在后台减少用户操作。

适合 Agent 做的事：

- 计划完成后生成下一阶段草案
- 同步后更新题库资产
- 检测需要复习的题
- 检测计划是否过难或过易
- 在必要时给一句短提示

不适合 Agent 做的事：

- 每天输出大段指导
- 替用户自动启用计划
- 自动改变学习方向
- 把简单任务变成复杂对话

---

## 12. 长期方向

LeetCoach 后续可以继续往三个方向发展。

### 12.1 更像正式软件

- 更现代的 UI
- 更顺滑的页面切换
- 更少弹窗
- 更强的状态反馈
- 更完整的本地数据保护

### 12.2 更懂用户的学习资产系统

- 每道题形成长期档案
- 复习安排基于真实掌握度
- AI 题解和用户笔记持续积累
- RAG 使用这些资产生成更个性化的解释和计划

### 12.3 更克制的学习 Agent

- Agent 在后台工作
- 只在关键节点提醒
- 不制造信息噪声
- 用户始终拥有最终确认权

最终目标不是做一个很吵的智能助手，而是做一个安静、可靠、长期可用的算法学习管家。
