# LeetCoach 本地模型实验总结

## 1. 实验目标

本阶段目标是学习和验证本地模型部署在 LeetCoach 中的可行性，而不是急于替换现有云端模型。

实验范围包括：

- 本地 Embedding 调用
- 本地 LLM 解题能力观察
- 云端 / 本地模型效果对比
- Embedding fallback 策略评估

通过这一阶段，LeetCoach 完成了从“只调用云端模型”到“能够接入、测试、评估本地模型”的工程学习闭环。

## 2. 本地 Embedding 结论

本地 Embedding 已经跑通。

当前实验模型：

```text
Ollama nomic-embed-text
```

当前结论：

- 可以通过 Ollama 正常生成向量。
- 向量维度为 768。
- 本地 Embedding 稳定性较好。
- 本地 Embedding 适合作为云端 Embedding 失败时的 fallback。
- 暂不建议直接替代云端 Embedding 默认策略。

原因是：虽然本地 Embedding 稳定性较好，但本地与云端 Top-K 检索排序仍存在明显差异。直接切换默认策略可能改变 RAG 检索结果，因此更稳妥的做法是保留云端优先，本地兜底。

## 3. 本地 LLM 解题结论

本地小模型可以生成简单题解释，但当前不适合正式接入 LeetCoach 的 AI 题解主流程。

主要原因：

- 响应速度较慢。
- 题解质量不够稳定。
- 输出过程性内容较多。
- 固定模板输出不够稳定。
- 对复杂题和完整代码正确性没有充分保障。

LeetCoach 的 AI 解题需要稳定输出：

1. 解题思路
2. 带详细注释的完整代码
3. 易错点总结
4. 时间复杂度

当前本地小模型在结构稳定性、代码可靠性和响应速度上还不足以承担正式题解生成任务。

## 4. 当前最终策略

当前模型策略如下：

- Embedding：云端优先，本地 fallback。
- AI 解题：云端模型为主。
- AI 计划：云端模型为主。
- 本地 LLM：仅保留为后续实验方向，不进入主流程。

这意味着 LeetCoach 当前不会把本地 LLM 接入正式 AI 解题，也不会把本地 Embedding 直接设为默认。

## 5. 后续何时重新考虑本地 LLM

只有在以下条件更成熟时，才建议重新评估本地 LLM：

- 本地更强模型可用。
- 推理速度能接受。
- JSON / 模板输出足够稳定。
- Easy / Medium 题解质量通过评估。
- 复杂题代码正确性有更可靠保障。
- 可以作为云端失败 fallback，而不是直接默认。

本阶段结论是：本地模型实验已经达到学习目的。下一步不应继续扩展本地模型实验功能，而应进入项目整理、展示和阶段总结。
